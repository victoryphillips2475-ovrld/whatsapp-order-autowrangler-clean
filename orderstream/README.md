# WhatsApp Order Auto‑Wrangler

**What it does**

A lightweight SaaS service that automates order intake for African micro‑retailers via WhatsApp Business.  Customers send a WhatsApp message with their order; the service creates a structured order in Appwrite, optionally sends a confirmation back to the customer, and exposes a merchant dashboard to view, confirm, fulfill, and export orders.

**Architecture**

- **Backend** – FastAPI (Python 3.12) offering a REST API (`/api/v1/*`).  Handles authentication (JWT), Appwrite integration, order CRUD, webhook handling, and health/readiness probes.
- **WhatsApp service** – Node 22 (Baileys) maintains a persistent WhatsApp Web session, receives inbound messages via a webhook, extracts order data and forwards it to the backend.
- **Static UI** – Vite‑based merchant dashboard and a Capacitor‑based mobile SPA bundled into `generated_ui/` and `mobile/dist/` respectively.
- **Reverse proxy** – Nginx serves the static UI and proxies `/api/*` to the FastAPI container.
- **Data store** – Appwrite (hosted cloud or self‑managed) stores users, orders and related collections.

**Prerequisites**

- Docker ≥ 27.0 and Docker‑Compose v2
- Python 3.12 (for local dev only)
- Node 22 (for local dev only)
- An Appwrite project with a database and collections for `users` and `orders`

**Quick start (production‑like)**

```bash
# 1. Copy the production env template and fill in real secrets
cp deploy/.env.production deploy/.env
# Edit deploy/.env and replace placeholder values (JWT_SECRET, APPWRITE_*, WEBHOOK_SECRET, etc.)

# 2. Build and start the stack
docker compose -f deploy/docker-compose.yml up -d --build
```

The stack will expose:
- `http://localhost` – Nginx serving the merchant dashboard (`/ui`) and mobile SPA (`/mobile`).
- `http://localhost/api/v1/...` – FastAPI JSON API (protected by JWT).
- Health endpoints: `http://localhost/api/v1/health` and `http://localhost/api/v1/ready`.

**Local development**

```bash

# WhatsApp service (watch mode)
cd orderstream/whatsapp
npm ci
npm run dev   # uses ./index.js and reads FASTAPI_URL from .env

# UI (Vite)
cd mobile
npm ci
npm run dev   # http://localhost:5173 – proxy to FastAPI via CORS
```

**Environment variables** – see `deploy/.env.production` for the full list.  The most important are:
- `JWT_SECRET` – secure random base64 string for signing JWTs.
- `APPWRITE_*` – endpoint, project ID, API key and database ID.
- `WEBHOOK_SECRET` – shared secret for authenticating Baileys webhook requests.
- `WEBHOOK_DEFAULT_USER_ID` – Appwrite user ID used for orders created by the webhook.
- `WHATSAPP_PHONE` – WhatsApp Business phone number (E.164 format).
